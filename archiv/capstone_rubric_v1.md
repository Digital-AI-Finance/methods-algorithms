# Capstone Project Rubric

**Course**: Methods and Algorithms - MSc Data Science
**Deliverable**: Written report (10-15 pages)
**Total Points**: 100

---

## Quick Grading Guide

For each criterion, circle the appropriate level. Sum the points for the final grade.
Typical grading time: 15 minutes per report.

---

## Criterion 1: Problem Definition (20 points)

| Level | Points | Description |
|-------|--------|-------------|
| Excellent | 20 | Clear, well-motivated business problem with specific success criteria |
| Good | 15 | Clear problem statement, adequate motivation |
| Adequate | 10 | Problem stated but vague or poorly motivated |
| Needs Work | 5 | Unclear problem, no business context |

**Score: _____ / 20**

**Checklist:**
- [ ] Business context explained
- [ ] Problem clearly stated
- [ ] Success criteria defined
- [ ] Relevance to course topics evident

---

## Criterion 2: Method Selection (20 points)

| Level | Points | Description |
|-------|--------|-------------|
| Excellent | 20 | Methods justified with alternatives considered, uses decision framework |
| Good | 15 | Methods justified, some comparison |
| Adequate | 10 | Methods stated but weak justification |
| Needs Work | 5 | No justification for method choice |

**Score: _____ / 20**

**Checklist:**
- [ ] At least 2 methods from course applied
- [ ] Justification for method selection
- [ ] Alternatives considered
- [ ] Decision framework reasoning used

---

## Criterion 3: Implementation (20 points)

| Level | Points | Description |
|-------|--------|-------------|
| Excellent | 20 | Correct implementation, efficient code, well-documented |
| Good | 15 | Correct implementation, adequate documentation |
| Adequate | 10 | Implementation works but has issues or poor documentation |
| Needs Work | 5 | Errors in implementation or no code shown |

**Score: _____ / 20**

**Checklist:**
- [ ] Code runs correctly
- [ ] Preprocessing appropriate
- [ ] Train/test split proper
- [ ] Hyperparameter tuning attempted
- [ ] Random seeds set for reproducibility

---

## Criterion 4: Results Interpretation (20 points)

| Level | Points | Description |
|-------|--------|-------------|
| Excellent | 20 | Business insights derived, limitations discussed, clear recommendations |
| Good | 15 | Good interpretation with some insights |
| Adequate | 10 | Basic interpretation, no actionable insights |
| Needs Work | 5 | No interpretation beyond reporting metrics |

**Score: _____ / 20**

**Checklist:**
- [ ] Results clearly presented
- [ ] Model comparison included
- [ ] Feature importance analyzed (if applicable)
- [ ] Business implications discussed
- [ ] Limitations acknowledged

---

## Criterion 5: Report Quality (20 points)

| Level | Points | Description |
|-------|--------|-------------|
| Excellent | 20 | Professional writing, well-structured, proper citations, polished |

**Score: _____ / 20**
